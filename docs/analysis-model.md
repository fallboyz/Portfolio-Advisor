# 분석 모델: 검증된 밸류에이션 지표 + Z-Score 합성

## 설계 원칙

개별 자산의 저평가/고평가 판단에는 학술적으로 검증된 밸류에이션 지표를 사용하고,
최종 실물 자산 vs ETF 비율 결정만 자체 로직으로 한다.

## Z-Score 개요

Z-Score = "현재 값이 평균에서 표준편차 몇 개만큼 떨어져 있는지"
- +2: 평균보다 표준편차 2개 위 (과열)
- -2: 평균보다 표준편차 2개 아래 (저평가)
- 0: 평균 수준

## 지표 선정 근거

### 금/은 (실물 자산)

**실질금리**: 금은 이자를 지급하지 않으므로 실질금리가 높을수록 금 보유의 기회비용이 커진다. 역사적으로 실질금리와 금 가격은 -0.8 이상의 역상관을 보여왔다. (World Gold Council, 다수 학술 연구)

**M2/금 비율**: 중앙은행의 통화 팽창(M2 증가)은 금의 구매력 보존 기능을 부각시킨다. M2 대비 금 가격이 낮으면 통화량 증가를 금 가격이 아직 반영하지 못한 상태로, 저평가 신호로 해석한다.

**GSR (금/은 비율)**: 금과 은의 상대가치를 나타내는 지표로, 귀금속 시장에서 수십 년간 사용되어 왔다. 장기 평균(60~80) 대비 높으면 은이 상대적으로 저평가, 낮으면 고평가로 판단한다.

### 주식 (ETF)

**CAPE (Shiller P/E)**: Robert Shiller 교수가 개발한 경기조정 주가수익비율. 10년 평균 실질 이익 대비 현재 주가를 평가한다. 1871년부터의 데이터로 장기 밸류에이션 판단에 가장 널리 사용되는 지표다. (Shiller, "Irrational Exuberance")

**Buffett Indicator (시총/GDP)**: Warren Buffett이 "가장 좋은 단일 밸류에이션 지표"라고 언급한 주식시장 시가총액 대 GDP 비율. 100%를 넘으면 주의, 200%를 넘으면 극단적 과평가 영역으로 본다.

**Yield Curve (장단기 금리차)**: 10년 국채금리와 3개월 국채금리의 차이. 역전(음수)은 경기 침체의 선행지표로 1960년대 이후 모든 미국 경기 침체를 예측했다. (NY Fed 연구)

## 가중치 설정 근거

가중치는 각 지표의 신뢰도와 데이터 가용성을 기준으로 설정했다.

**금/은**: 실질금리(30%)와 M2/금(30%)이 핵심. 이 둘은 금의 펀더멘털 가치를 가장 직접적으로 반영하는 매크로 지표다. 가격 위치(20%)와 수익률(20%)은 모멘텀/평균회귀 보조 지표.

**S&P500**: CAPE(35%)가 가장 긴 역사(150년+)와 학술적 검증을 갖고 있어 최고 가중치. Buffett(25%)은 시장 전체 규모를 반영. Yield Curve(15%)는 경기 방향성 신호로 보조적. 수익률(25%)은 모멘텀 포착.

**나스닥100**: CAPE/Buffett 데이터가 나스닥 전용으로 없어서 수익률(10년 40%, 5년 40%)과 가격 위치(20%)로 구성. 데이터 제약에 따른 차선책.

**은**: 금과 동일한 매크로 지표(실질금리 25%, M2/금 25%)에 GSR(20%)을 추가. 은은 금과 달리 산업 수요가 크고 변동성이 높아 GSR로 상대가치를 별도 평가.

> 모든 가중치는 config.toml에서 조정 가능하다.

## 사용 지표

### 검증된 밸류에이션 지표 (주요)

| 지표 | 대상 | 데이터 소스 | 의미 |
|------|------|-----------|------|
| 10년 실질금리 | 금/은 | FRED (REAINTRATREARAT10Y) | 실질금리 높을수록 금에 불리 (역상관). Z-Score 부호 반전 적용 |
| M2/금 비율 | 금/은 | FRED (M2SL) / 금 가격 | 통화량 대비 금 가격. 높을수록 금이 저평가 |
| CAPE (Shiller P/E) | S&P500 | Shiller Excel | 경기조정 P/E. 높을수록 주식 과평가. 전체 역사(1871~) 대비 |
| Buffett Indicator | S&P500 | FRED (GDP) / S&P500 | 시총/GDP. 높을수록 주식 과평가 |
| Yield Curve | S&P500 | FRED (DGS10 - DGS3MO) | 장단기 금리차. 음수(역전) = 경기 둔화 신호 |
| GSR (금/은 비율) | 은 | 금/은 가격 | 비율 높으면 은 저평가, 낮으면 은 고평가 |

### 보조 지표

| 지표 | 대상 | 의미 |
|------|------|------|
| 가격 위치 | 금/은/나스닥 | 10년 이동평균 대비 현재 가격 위치 |
| 수익률 Z-Score (10년) | 전체 | 10년 수익률 역사 분포 대비 현재 위치 |
| 수익률 Z-Score (5년) | 나스닥 | 5년 수익률 역사 분포 대비 (50년 데이터 부족) |

## 복합 점수 산출

각 자산별로 검증된 지표가 주요 입력(60~70%), 보조 지표가 나머지를 차지.

**금 복합 점수 (config.toml `[weights_gold]`):**
```
S_gold = 0.30*Z_real_rate + 0.30*Z_m2_gold + 0.20*Z_price_position + 0.20*Z_return_10y
```
- 실질금리 Z-Score는 부호 반전 (높은 금리 = 금에 불리)

**은 복합 점수 (config.toml `[weights_silver]`):**
```
S_silver = 0.25*Z_real_rate + 0.25*Z_m2_gold + 0.15*Z_price_position + 0.15*Z_return_10y + 0.20*Z_gsr
```

**S&P500 복합 점수 (config.toml `[weights_sp500]`):**
```
S_sp500 = 0.35*Z_cape + 0.25*Z_buffett + 0.15*Z_yield_curve + 0.25*Z_return_10y
```

**나스닥100 복합 점수 (config.toml `[weights_ndx]`):**
```
S_ndx = 0.40*Z_return_10y + 0.40*Z_return_5y + 0.20*Z_price_position
```

> 일부 Z-Score가 데이터 부족으로 계산 불가한 경우, 사용 가능한 항목만으로 자동 renormalization.

## 2단계 비율 조정 신호

### Level 1: 실물 자산 vs ETF

**그룹 점수:**
```
S_precious = (S_gold + S_silver) / 2
S_etf = S_sp500 * 0.7 + S_ndx * 0.3
R_group = S_precious - S_etf
```

| R_group 구간 | 신호 | 실물 자산 비중 |
|-------------|------|-------------|
| R < -2.0 | 실물 대폭 확대 | 75% |
| -2.0 ~ -1.0 | 실물 확대 | 60% |
| -1.0 ~ +1.0 | 균형 | 50% |
| +1.0 ~ +2.0 | ETF 확대 | 35% |
| R > +2.0 | ETF 대폭 확대 | 20% |

### Level 2a: 금 vs 은

Gold/Silver Ratio Z-Score 기반:

| GSR Z-Score | 금:은 비율 |
|------------|-----------|
| Z > 1.0 | 30:70 (은 크게 저평가) |
| 0 ~ 1.0 | 40:60 |
| -1.0 ~ 0 | 50:50 |
| Z < -1.0 | 60:40 (금이 더 매력적) |

### Level 2b: S&P 500 vs 나스닥 100

상대 복합 점수 차이 기반:

| S_sp500 - S_ndx | S&P:나스닥 비율 |
|-----------------|---------------|
| 차이 > 0.5 | 40:60 (나스닥 저평가) |
| -0.5 ~ 0.5 | 50:50 |
| 차이 < -0.5 | 60:40 (S&P 저평가) |

## 폭락/폭등 보정 (Drawdown/Rally Overlay)

| 조건 | 보정 |
|------|------|
| 은 고점 대비 -40% 이하 | 실물 점수에 -0.5 (더 매력적) |
| ETF 고점 대비 -30% 이하 | ETF 점수에 -0.5 (더 매력적) |
| 은 6개월 +100% 이상 | 실물 점수에 +0.5 (과열 경고) |

## Z-Score 부호 방향 해석

- 점수가 높은(양수) 자산 = 고평가 가능성
- 점수가 낮은(음수) 자산 = 저평가 가능성
- R이 음수: 실물이 저평가, ETF가 고평가 -> 실물 비중 올림
- R이 양수: 실물이 고평가, ETF가 저평가 -> ETF 비중 올림
- 실질금리 Z-Score는 부호 반전 적용 (높은 금리 = 금 불리이므로)

## MCP 분석 워크플로우

자동 분석(스케줄러)은 숫자 계산만 수행. MCP를 통한 AI 분석 시 아래 워크플로우를 따른다.

### 데이터 수집 단계

1. `get_scores` - Z-Score + 복합 점수 데이터
2. `get_signals` - R 점수, 신호, 비율
3. `get_history` - 자산별 최근 가격 추이
4. `get_report` - 전체 리포트 (신호 + 코멘트 + 데이터 상태)
5. `get_news` - 자산별 최근 뉴스 (Finnhub API 기반)

### 뉴스 활용 규칙

- 지표와 직접 연결되는 뉴스만 사용
- 매크로/정책 같은 광범위한 원인 추적은 하지 않음
- 가격 변동의 원인은 이미 가격에 반영되어 있으므로 원인 역추적 불필요
- 뉴스 API에서 구조화된 데이터로만 참조 (AI가 직접 웹 검색하지 않음)

### 코멘트 작성 규칙

**데이터 분석 (시스템 자동 생성 + AI 보충)**
- 비율, R 점수, 자산별 점수, 주요 Z-Score 수치 나열
- 검증된 지표 수치를 명시적으로 포함

**시장 동향**
- 자산당 1~2줄, 핵심 사실만
- 센티먼트 스코어가 높은 뉴스만 언급
- 지표 수치와 연결해서 작성 (예: "Buffett Z=+3.18인 가운데 주식시장은...")
- 감정적 표현 금지 (사실과 숫자 기반)

**종합 판단**
- 3~5줄, R 점수 결과를 뒷받침하는 근거 위주
- "왜 이 비율이 나왔는가"에 대한 답

**전체**
- 데이터 분석 + 시장 동향 + 종합 판단 합쳐서 15줄 이내
- `add_comment`로 DB에 저장 -> 대시보드에 반영
